Alapvető SQL Kérdések

  1. Mi az SQL?

    Az SQL (Structured Query Language) egy adatbázis-kezelő nyelve, amelyet az adatok lekérdezésére és manipulálására használnak.

  2. Mik a leggyakoribb SQL utasítások?

    A leggyakoribb SQL utasítások közé tartozik a SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER és DROP.

  3. Mi a különbség a WHERE és a HAVING között?

    A WHERE a sorok szűrésére szolgál a GROUP BY előtt, míg a HAVING a csoportosított adatok szűrésére használható a GROUP BY után.

  4. Mi az a PRIMARY KEY?

    A PRIMARY KEY egy vagy több oszlop, amely egyedileg azonosítja a rekordokat egy táblában, és nem tartalmazhat NULL értékeket.

  5. Mi az a subquery, és mikor használják?

    A subquery (al-lekérdezés) egy lekérdezés, amely egy másik lekérdezésen belül található. Gyakran használják összetett lekérdezésekhez, ahol egy lekérdezés eredményére van szükség a másikban.

  6. Hogyan használhatók az aggregáló függvények SQL-ben?

    Az aggregáló függvények, mint például COUNT(), SUM(), AVG(), MAX(), és MIN(), az adatok csoportosítására és összesítésére szolgálnak. Ezeket a GROUP BY klauzulával együtt használják.

  7. Mi az a NULL érték, és hogyan kezelhető az SQL-ben?

    A NULL érték az adat hiányát jelzi. Az SQL-ben a NULL értékeket az IS NULL vagy IS NOT NULL operátorokkal kezeljük.

  8. Hogyan kérdeznéd le a legmagasabb árú terméket egy termékek táblájából?

    SELECT * FROM products ORDER BY price DESC LIMIT 1;

További alapfogalmak

  • Mit nevezünk adatbázisnak?
    Egy strukturált adatgyűjtemény, amelyben az adatok rendszerezett formában, táblákban vannak tárolva.
  • Mit nevezünk adattáblának?
    Az adattábla egy adatbázison belüli struktúra, amely sorokat (rekordokat) és oszlopokat (mezőket) tartalmaz.
  • Hogyan kapcsolódhatnak egymáshoz?
    Az adattáblák közötti kapcsolatok általában kulcsok segítségével jönnek létre. Ezek a kapcsolatok lehetnek:
    1. Egy-egy (One-to-One): Egy rekord csak egy másik rekordhoz kapcsolódik egy másik táblában.
    2. Egy-több (One-to-Many): Egy rekord több rekordhoz kapcsolódik egy másik táblában.
    3. Több-több (Many-to-Many): Több rekord több rekordhoz kapcsolódik egy másik táblában, amelyhez kapcsolótáblára van szükség.
  • Mit nevezünk mezőnek, rekordnak, adatnak?
    Mező: az adattábla egy oszlopa (pl. név, életkor).
    Rekord: az adattábla egy sora, amely egy entitás adatait tartalmazza.
    Adat: az egyes mezők értékei a rekordokon belül.
  • Milyen kulcsokat ismersz? Ismertesd őket!
    • PRIMARY KEY: Egyedi azonosító minden rekordhoz, nem lehet NULL.
    • FOREIGN KEY: Másik tábla elsődleges kulcsára hivatkozik, kapcsolatot teremt.
    • UNIQUE: Biztosítja, hogy az értékek egyediek legyenek egy oszlopban.
  • Mit nevezünk kapcsolótáblának?
    Egy olyan táblát, amely két másik tábla közötti több-a-többhöz kapcsolatot valósít meg, gyakran két idegen kulcsot tartalmaz.
  • Milyen adattípusokat használunk adatbázis-kezelőben?
    • Szöveg típusok: VARCHAR, TEXT
    • Szám típusok: INT, FLOAT, DOUBLE
    • Dátum/idő: DATE, TIME, DATETIME
    • Logikai: BOOLEAN

Mintatétel kérdésekkel:

Tétel: SQL utasítások és függvények

  1. Ismertesd az adatbázis létrehozásához szükséges utasítást!

    Az adatbázis létrehozásához az SQL CREATE DATABASE utasítást használjuk.

    Szintaxis:
    CREATE DATABASE adatbazis_neve;

    Példa:
    CREATE DATABASE tanuloi_adatbazis;

  2. Ismertesd a táblák importálásához szükséges SQL utasításokat!

    A táblák adatainak importálására különböző módszerek léteznek, a leggyakoribbak az INSERT INTO utasítás használata vagy külső fájlok betöltése speciális SQL parancsokkal.

    INSERT INTO szintaxis:
    INSERT INTO tabla_neve (oszlop1, oszlop2, ...) VALUES (ertek1, ertek2, ...);

    Példa:
    INSERT INTO hallgatok (hallgato_id, nev, születési_dátum) VALUES (1, 'Kiss Péter', '2000-04-15');

    Külső fájlok importálására adatbázis-kezelő függően használható például a LOAD DATA INFILE (MySQL), vagy az adatbázis adminisztrációs eszközeiben elérhető import funkciók.

  3. Mutasd be a táblák bővítéséhez, frissítéséhez, törléséhez szükséges utasításokat!

    Táblák bővítése (oszlop hozzáadása):
    Az oszlop hozzáadásához az ALTER TABLE utasítást használjuk.

    Szintaxis:
    ALTER TABLE tabla_neve ADD oszlop_neve adat_tipus;

    Példa:
    ALTER TABLE hallgatok ADD email VARCHAR(255);

    Adatok frissítése:
    Az adatok módosítására a UPDATE utasítást alkalmazzuk.

    Szintaxis:
    UPDATE tabla_neve SET oszlop_neve = új_érték WHERE feltétel;

    Példa:
    UPDATE hallgatok SET email = 'kiss.peter@example.com' WHERE hallgato_id = 1;

    Adatok törlése:
    Az adatok törléséhez a DELETE FROM utasítást használjuk.

    Szintaxis:
    DELETE FROM tabla_neve WHERE feltétel;

    Példa:
    DELETE FROM hallgatok WHERE hallgato_id = 1;

  4. Ismertesd a lekérdezések szintaxisát!

    Az SQL lekérdezések alapvető szerkezete a SELECT utasításon alapul, amely az adatbázisból adatokat kérdez le.

    Szintaxis:
    SELECT oszlop1, oszlop2, ... FROM tabla_neve WHERE feltétel GROUP BY oszlop HAVING feltétel ORDER BY oszlop LIMIT szám;

    Példa:
    SELECT nev, születési_dátum FROM hallgatok WHERE születési_dátum > '2000-01-01' ORDER BY nev ASC;

  5. Milyen aggregáló függvényeket ismersz?

    Az SQL leggyakoribb aggregáló függvényei:

    • COUNT() – rekordok számának megszámlálása
    • SUM() – értékek összegzése
    • AVG() – értékek átlagolása
    • MAX() – legnagyobb érték
    • MIN() – legkisebb érték
  6. Milyen dátum- és időfüggvényeket ismersz?

    SQL-ben különböző dátum- és időfüggvények állnak rendelkezésre, például:

    • NOW() – aktuális dátum és idő
    • CURRENT_DATE() – aktuális dátum
    • CURRENT_TIME() – aktuális idő
    • DATE() – dátum kivonása időbélyegből
    • DATEDIFF() – két dátum közti napok száma
    • DATE_ADD(), DATE_SUB() – dátumhoz hozzáadás, vagy levonás
  7. Mit tudsz a szövegek kezeléséről SQL-ben?

    SQL-ben különféle függvények segítik a szöveges adatok kezelését:

    • CONCAT() – szövegek összefűzése
    • SUBSTRING() vagy SUBSTR() – szöveg egy részének kivágása
    • LENGTH() – szöveg hosszának meghatározása
    • UPPER() és LOWER() – szöveg nagy- vagy kisbetűssé alakítása
    • TRIM() – szóközök eltávolítása a szöveg elejéről és végéről
  8. Mit tudsz az allekérdezésekről (subqueries)?

    Az allekérdezés egy olyan SQL lekérdezés, amely egy másik lekérdezésen belül található, és eredménye felhasználható a külső lekérdezésben.

    Használatuk lehetőséget ad összetett adatkiválasztási műveletek elvégzésére, például feltételek ellenőrzésére vagy aggregált adatok beillesztésére.

    Példa:
    SELECT nev FROM hallgatok WHERE hallgato_id IN (SELECT hallgato_id FROM jegyek WHERE tantargy = 'Matematika' AND jegy = 5);

SQL JOIN típusok

Az SQL JOIN műveletek célja, hogy két vagy több táblából adatokat kapcsoljanak össze közös mezők alapján.

Ezt írásbelin elegendő volt megadni a WHERE záradékban, viszont szóbelin rákérdezhetnek.

INNER JOIN

Az INNER JOIN csak azokat a rekordokat adja vissza, ahol mindkét táblában van egyező érték a csatlakoztatott mezőkön. Ha nincs egyezés, a sorok nem jelennek meg az eredményben.

LEFT JOIN

A LEFT JOIN visszaadja a bal oldali tábla összes rekordját, valamint a jobb oldali tábla egyező sorait. Ha a jobb oldalon nincs megfelelő egyezés, akkor az eredményben a jobb oldali tábla mezői NULL értéket kapnak.

RIGHT JOIN

A RIGHT JOIN visszaadja a jobb oldali tábla összes rekordját, valamint a bal oldali tábla egyező sorait. Ha a bal oldalon nincs megfelelő egyezés, akkor az eredményben a bal oldali tábla mezői NULL értéket kapnak.
